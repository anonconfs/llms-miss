# Claude PR Review GitHub Action

This GitHub Action automatically reviews pull requests using Claude AI (Anthropic's API) whenever a PR is created or updated.

## Features

- ?? Automatic AI-powered code review on PR creation and new commits
- ?? Reviews based on your project's coding standards (from `AGENTS.md`)
- ?? Provides feedback on code quality, architecture, potential issues, and best practices
- ? Posts review as a PR comment
- ?? Triggers on PRs to `main` and `Update/**` branches

## Setup Instructions

### 1. Get an Anthropic API Key

1. Go to [Anthropic Console](https://console.anthropic.com/)
2. Sign up or log in to your account
3. Navigate to API Keys section
4. Create a new API key
5. Copy the key (it starts with `sk-ant-`)

### 2. Add API Key to GitHub Secrets

1. Go to your GitHub repository
2. Click on **Settings** ? **Secrets and variables** ? **Actions**
3. Click **New repository secret**
4. Name: `ANTHROPIC_API_KEY`
5. Value: Paste your Anthropic API key
6. Click **Add secret**

### 3. Enable Workflow

The workflow is already configured in `.github/workflows/claude-pr-review.yml`. It will automatically run when:
- A new pull request is opened
- New commits are pushed to an existing pull request
- PRs targeting `main` or any `Update/**` branch

### 4. Permissions

The workflow needs the following permissions (already configured):
- `contents: read` - To access repository code
- `pull-requests: write` - To post review comments

## How It Works

1. **Trigger**: When a PR is created or updated
2. **Fetch Changes**: Gets the diff between base and PR branch
3. **Context Gathering**: Reads project context from `AGENTS.md`
4. **AI Review**: Sends the diff and context to Claude for review
5. **Post Comment**: Adds Claude's review as a comment on the PR

## Review Criteria

Claude reviews your code based on:
- **Code Quality**: Adherence to project coding standards
- **Architecture**: Compatibility with established patterns (DI, service extensions)
- **Potential Issues**: Bugs, edge cases, and problems
- **Best Practices**: .NET 10 best practices
- **Performance**: Performance concerns
- **Testing**: Test coverage recommendations
- **Positive Feedback**: What's done well

## Customization

### Change Claude Model

Edit the `model` field in `.github/workflows/claude-pr-review.yml`:
```javascript
model: 'claude-sonnet-4-20250514'
```

**Note:** Use the model specified above. AI agents with older knowledge cutoffs may suggest outdated model names that will result in 404 errors.

### Adjust Max Tokens

Modify `max_tokens` to control review length:
```yaml
"max_tokens": 4096
```

### Change Trigger Branches

Edit the `branches` section:
```yaml
on:
  pull_request:
    types: [opened, synchronize]
    branches:
      - main
      - 'Update/**'
      - develop  # Add more branches
```

### Customize Review Prompt

Modify the prompt in the `Request Claude review` step to adjust what Claude focuses on.

## Cost Considerations

Claude API pricing (as of 2024):
- Claude 3.5 Sonnet: ~$3 per million input tokens, ~$15 per million output tokens
- Typical PR review: 1,000-5,000 tokens input, 500-2,000 tokens output
- Estimated cost per review: $0.01-0.05

For high-activity repositories, consider:
- Limiting to specific branches
- Only triggering on `opened` (not `synchronize`)
- Using Claude Haiku for faster, cheaper reviews

## Troubleshooting

### Review Not Posted
- Check that `ANTHROPIC_API_KEY` is correctly set in GitHub Secrets
- Verify the API key is active in Anthropic Console
- Check the Actions tab for error logs

### API Rate Limits
- Anthropic has rate limits based on your tier
- If exceeded, the workflow will fail
- Consider reducing triggers or upgrading your Anthropic plan

### Large Diffs
- Very large PRs may exceed token limits
- Consider splitting large PRs into smaller ones
- Increase `max_tokens` if reviews are cut off

## Disabling the Workflow

To temporarily disable:
1. Go to **Actions** ? **Workflows**
2. Select "Claude PR Review"
3. Click "Disable workflow"

Or delete the file `.github/workflows/claude-pr-review.yml`
