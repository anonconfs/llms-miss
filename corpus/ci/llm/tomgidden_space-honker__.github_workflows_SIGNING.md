# Code Signing and Notarization Setup

GitHub Actions workflow has been generated by Claude Sonnet 4.5, but seems sound. It included the instructions below, but at the moment the repo is configured for ad-hoc signing only.

## Quick Start

**The workflow works without any setup!** By default, it builds with ad-hoc signing (unsigned) which is perfect for open source development and testing. The DMG will be labeled as "Unsigned" in the artifacts.

To enable proper code signing and notarization (for distribution), follow the steps below to configure the required secrets.

## Prerequisites

1. **Apple Developer Account** - Required for code signing and notarization
   - Enroll at https://developer.apple.com/programs/
   - Cost: $99/year

2. **Developer ID Certificate** - For distribution outside the App Store
   - Type: "Developer ID Application"

## Step 1: Create Developer ID Certificate

1. Open **Keychain Access** on your Mac
2. Go to **Keychain Access > Certificate Assistant > Request a Certificate From a Certificate Authority**
3. Enter your email and name, select **"Saved to disk"**
4. Save the certificate request file

5. Go to https://developer.apple.com/account/resources/certificates/list
6. Click **+** to create a new certificate
7. Select **Developer ID Application** and click **Continue**
8. Upload your certificate request file
9. Download the certificate and double-click to install it in Keychain

## Step 2: Export Certificate for GitHub

1. Open **Keychain Access**
2. Find your **Developer ID Application** certificate
3. Expand it to show the private key
4. Select both the certificate and private key
5. Right-click and choose **Export 2 items...**
6. Save as a `.p12` file with a strong password (you'll need this later)

7. Convert the `.p12` file to base64:
   ```bash
   base64 -i YourCertificate.p12 | pbcopy
   ```
   This copies the base64 string to your clipboard

## Step 3: Create App-Specific Password

1. Go to https://appleid.apple.com/account/manage
2. Sign in with your Apple ID
3. In the **Security** section, click **Generate Password** under App-Specific Passwords
4. Enter a label like "GitHub Actions Notarization"
5. Copy the generated password (format: `xxxx-xxxx-xxxx-xxxx`)

## Step 4: Find Your Team ID

1. Go to https://developer.apple.com/account
2. Click on **Membership** in the sidebar
3. Your **Team ID** is shown (10 characters, e.g., `ABCDE12345`)

## Step 5: Configure GitHub Secrets

Go to your GitHub repository settings: **Settings > Secrets and variables > Actions**

Add the following secrets:

| Secret Name | Description | Example |
|-------------|-------------|---------|
| `BUILD_CERTIFICATE_BASE64` | Base64-encoded .p12 certificate | `MIIKvgIBAzCCCn...` |
| `P12_PASSWORD` | Password for the .p12 certificate | `YourStrongPassword` |
| `KEYCHAIN_PASSWORD` | Temporary keychain password (any strong password) | `TempPassword123!` |
| `CODE_SIGN_IDENTITY` | Certificate common name | `Developer ID Application: Your Name (TEAMID)` |
| `TEAM_ID` | Your Apple Developer Team ID | `ABCDE12345` |
| `APPLE_ID` | Your Apple ID email | `your@email.com` |
| `APPLE_ID_PASSWORD` | App-specific password from Step 3 | `xxxx-xxxx-xxxx-xxxx` |

### Finding Your CODE_SIGN_IDENTITY

Run this command to find your certificate name:
```bash
security find-identity -v -p codesigning
```

Look for the line with "Developer ID Application" and copy the full name in quotes, e.g.:
```
Developer ID Application: John Doe (ABCDE12345)
```

## Step 6: Update Xcode Project Settings

The project should be configured to use manual code signing. Verify in Xcode:

1. Select the project in the navigator
2. Select the **Space Honker** target
3. Go to **Signing & Capabilities** tab
4. Uncheck **Automatically manage signing**
5. Set **Team** to your development team
6. Set **Signing Certificate** to your Developer ID certificate

## Step 7: Test the Workflow

1. Commit and push your changes:
   ```bash
   git add .github/workflows/build.yml
   git commit -m "Add CI/CD with code signing"
   git push
   ```

2. Go to your GitHub repository's **Actions** tab
3. Watch the build process
4. Download the notarized DMG from the artifacts

## Troubleshooting

### Build fails with "User interaction is not allowed"
- Check that `KEYCHAIN_PASSWORD` is set correctly
- Ensure the keychain partition list command runs successfully

### Notarization fails
- Verify `APPLE_ID` and `APPLE_ID_PASSWORD` are correct
- Check that the app-specific password hasn't expired
- Ensure `TEAM_ID` matches your developer account

### Certificate not found
- Check that `CODE_SIGN_IDENTITY` exactly matches the certificate name
- Verify the certificate was imported successfully in the workflow logs

### Notarization takes too long
- Normal notarization takes 2-5 minutes
- The workflow waits up to 30 minutes by default
- Check notarization status at https://developer.apple.com/

## Security Notes

- **Never commit certificates or passwords to your repository**
- GitHub Secrets are encrypted and only exposed to authorized workflows
- The temporary keychain is deleted after each build
- Use different passwords for P12 and keychain
- Rotate app-specific passwords periodically

## Alternative: Skip Code Signing for Open Source

If you don't want to set up code signing (for open source development):

1. Remove the code signing steps from `.github/workflows/build.yml`
2. Build with ad-hoc signing:
   ```yaml
   CODE_SIGN_IDENTITY="-"
   CODE_SIGN_STYLE=Automatic
   ```

Note: Users will see a warning when opening the app without proper code signing.

## Creating Releases

To create a notarized release:

1. Tag your commit:
   ```bash
   git tag -a v0.2 -m "Release version 0.2"
   git push origin v0.2
   ```

2. Go to GitHub: **Releases > Draft a new release**
3. Select your tag
4. The workflow will automatically upload the notarized DMG
